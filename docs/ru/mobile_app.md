# Мобильное приложение (Android / Flutter)

Клиентское приложение ByteAway спроектировано для обеспечения высокой производительности и удобного пользовательского опыта. Оно совмещает в себе гибкий кроссплатформенный интерфейс на Flutter и низкоуровневые сетевые возможности ОС Android.

## Слой Flutter (UI и Бизнес-логика)

Исходный код расположен в каталоге `android/lib/`.

### Основные компоненты и структура
- **Маршрутизация (`app/router.dart`)**: Навигация реализована на базе пакета `go_router`. Включает в себя проверку авторизации (Auth Guard) и вложенные маршруты `StatefulShellRoute` для плавной смены вкладок (Главный экран, Статистика, Настройки).
- **Управление состоянием (`presentation/*/`)**: Архитектура использует паттерн BLoC/Cubit. Ключевые кубиты:
  - `VpnCubit` – Отвечает за запуск, остановку и получение текущего статуса VPN-подключения.
  - `SettingsCubit` – Управляет настройками пользователя, лимитами скорости и выбором транспорта.
  - `AuthCubit` – Отвечает за хранение токенов авторизации и статус сессии.
- **Работа с данными (`data/repositories/`)**: Инкапсулирует вызовы к удаленному API (`ApiClient`) и взаимодействие с нативными платформами через репозиторий `VpnRepositoryImpl`.

### Платформенные каналы (Platform Channels)

Платформенные каналы обеспечивают передачу данных между Dart и нативным кодом Android:

- **`com.byteaway.service` (MethodChannel)**: Принимает команды управления `startVpn`, `stopVpn`, `startNode`, `stopNode`, `getStatus`.
- **`com.byteaway.service/events` (EventChannel)**: Передает в реальном времени поток обновлений статуса, включая скорость передачи и возможные ошибки.
- **`com.ospab.byteaway/app` (MethodChannel)**: Отвечает за управление списками приложений для раздельного туннелирования: `getInstalledApps`, `getExcludedApps`, `addExclude`, `removeExclude`.

## Нативный слой (Kotlin & C/Go)

Исходный код расположен в каталоге `android/android/app/src/main/kotlin/com/ospab/byteaway/`.

### 1. `ServiceBridge.kt`
Выступает в роли центрального моста для приема вызовов из Flutter-слоя:
- Запрашивает разрешения на создание VPN-туннеля через системный диалог `VpnService.prepare()`.
- Формирует и передает команды (`Intents`) в службу `ByteAwayForegroundService`.
- Транслирует входящие события о скорости и состоянии сервиса обратно в Dart.

### 2. `ByteAwayForegroundService.kt`
Фоновая служба (Foreground Service), работающая независимо от жизненного цикла графического интерфейса:
- **Управление VPN**: Создает и конфигурирует TUN-интерфейс с помощью системного `VpnService.Builder`. Устанавливает IP-адрес `172.19.0.1`, MTU и DNS-серверы. Применяет правила `addDisallowedApplication`.
- **Предотвращение петель трафика**: Пакет `com.ospab.byteaway` всегда добавляется в список исключений для корректного обмена трафиком с Мастер-нодой при включенном VPN.
- **Интеграция Node**: Запускает и поддерживает соединение (QUIC или WebSocket) с сервером для передачи клиентского трафика.

### 3. Интеграция Sing-Box Go
Сетевые операции осуществляются через Go-библиотеку `boxwrapper.aar`, скомпилированную при помощи `gomobile`.
- Входящий сетевой трафик передается напрямую из Android TUN-интерфейса в ядро `sing-box` через метод `Boxwrapper.startSingBox(jsonConfig)`.
- Конфигурационный JSON генерируется динамически на основе активного протокола (VLESS или OSTP) и текущих ключей шифрования.

## Особенности сборки и релиза

- **Скрипт автоматизации**: Сборка осуществляется через PowerShell скрипт `build.ps1`, который компилирует Go-модуль, собирает Flutter в релизный APK и подготавливает манифест обновлений.
- **Обфускация и ProGuard**: Правила обфускации зафиксированы в файле `proguard-rules.pro`. Все классы `boxwrapper.**` защищены от удаления и переименования (`-keep class boxwrapper.** { *; }`) для предотвращения сбоев при вызовах через JNI.
